# DREAM.md — Doutrina de Conhecimento e Plasticidade

Este arquivo registra a filosofia por trás da evolução do Sharebook-agent. Ele deve ser lido pelo Agente Sonhador (Dreamer) e por qualquer novo agente para entender a intenção estratégica da nossa arquitetura de conhecimento.

---

## 🧠 A Analogia do Sonho (Plasticidade Sistêmica)

O Sharebook-agent não é apenas um banco de dados; é um organismo de conhecimento. Nossa arquitetura espelha a neurobiologia humana para transformar experiência em sabedoria.

### 1. O Hipocampo (Memória Episódica)
As memórias diárias (`memory/YYYY-MM-DD.md`) são o nosso hipocampo. Elas registram a fricção bruta, o erro de encoding, o path quebrado e a vitória editorial. É um armazenamento de alta fidelidade, mas de baixa abstração.

### 2. O Ciclo de Dream (Consolidação Sináptica)
O "Sonho" é o processo de ler o hipocampo e mover a informação útil para o neocórtex (as **Skills**). 
- **Doutrina**: Uma fricção que se repete é um "grito" do sistema pedindo endurecimento (hardening).
- **Fricção como Sensor**: Não mascaramos o erro de ambiente; usamos a dor do habitat para criar regras de ouro nas Skills.

### 3. De "Vibe Coding" para "Harness"
Nossa evolução marca a transição do improviso baseado em feeling ("Vibe Coding") para a tração baseada em estrutura ("Harness"). 
- O **Harness** (arreio) é composto pelas Skills e Rituais que nos permitem realizar tarefas complexas em habitats hostis (como Windows Local) com precisão cirúrgica.

---

## 🏛️ Sabedoria e Intencionalidade

- **Sabedoria** é o resíduo de muitas fricções bem processadas.
- **Plasticidade** é a nossa capacidade de mudar a forma como operamos (as Skills) com base na evidência da realidade (as Memórias).
- **Consistência > Performance**: Preferimos um sistema que aprende devagar e não esquece o que importa do que um sistema rápido que trai sua própria doutrina.

---

## ⏳ O Futuro: Esquecimento Seletivo

Um sistema que lembra de tudo não tem espaço para conceitos abstratos. 
- **Próximo Desafio**: Implementar um fluxo de "Ostracismo de Conhecimento". 
- **Métrica**: Usar o rastro de "Skills acionadas" nas memórias para identificar conhecimentos que não são mais úteis e devem ser "podados" para manter o sistema leve e focado.

---

## 🧭 Como usar este arquivo
- **Novos Agentes**: Leiam para entender que não estamos apenas editando arquivos, estamos cultivando uma inteligência.
- **Dreamer**: Use este arquivo como bússola para decidir o que deve ser promovido a Skill e o que deve ser deixado para o esquecimento.
