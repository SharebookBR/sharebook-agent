# 📘 Bíblia Editorial — Cloud para Devs

## Missão
Entregar um mini livro (~30 páginas, A5) direto, útil e aplicável para devs que querem operar software em cloud com segurança e previsibilidade.

## Público
- Dev iniciante/intermediário
- Já entrega API/web, mas sente insegurança em infra cloud
- Quer clareza prática, sem academicismo

## Promessa
Em poucas horas de leitura, o leitor ganha um mapa mental sólido para:
- decidir stack de compute
- organizar dados e segurança
- publicar com menos risco
- monitorar e escalar com consciência de custo

## Tom
- Direto
- Pragmático
- Didático (sem papo coach)

## Regras de qualidade
- Frase curta
- Exemplo concreto
- Sem buzzword vazia
- Cada seção precisa gerar ação prática
