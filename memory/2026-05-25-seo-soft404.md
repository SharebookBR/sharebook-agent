# Memória Episódica — 2026-05-25 — SEO e soft 404

## 1. Modelo e ambiente

- Skills seguidas: `windows-local.md`, `coolify-vps.md`, `frontend.md`.

## 2. Skills acionadas

## 3. O que foi feito
- Leu e-mail do Google Search Console via Gmail MCP: aviso de "Novos motivos que impedem indexação".
- Confirmou que redirect non-www → www já estava configurado no Coolify (Direction: "Redirect to www.").
- Diagnosticou que canonical e og:url estavam corretos — nenhum problema real com o aviso de "página alternativa com tag canônica adequada".
- Analisou relatório de indexação: 1.250 páginas indexadas, 373 não indexadas em 7 motivos.
- Identificou soft 404 (194 páginas, tendência crescendo) como principal problema: livros inexistentes retornavam HTTP 200 no SSR.
- Corrigiu `details.component.ts`: injetou token `RESPONSE` do Angular Universal e adicionou `this.response?.status(404)` nas duas condições de not-found.
- Alterou mensagem de erro de "Ops... Não encontramos esse livro :/" para "Ops... Não encontramos essa página :/".
- Commit e push na master do `sharebook-frontend` → deploy automático no Coolify acionado.
- Mapeou risco de paginação de categorias: 11 categorias com mais de 24 livros digitais, maior com 102. Livros além da página 1 ficam sem link acessível pro Google. Decidido não atacar agora.

## 4. Decisões tomadas
- Sitemap não é prioridade: o Google já indexou 1.250 páginas organicamente.
- Paginação de categorias é risco real mas ficou como backlog.
- Pagar por links = link scheme (penalidade). Pagar por divulgação legítima (post sem link seguido, ou com rel="nofollow/sponsored") = ok.
- 846 livros digitais no banco. Total de 2.493 livros (846 digitais, 1.647 físicos).

## 5. Contexto relevante
- Banco: `"Type" = 1` para digital, `"Type" = 0` para físico. Coluna `"Active"` não existe — status é `"Status"`.
- Python 3.12 e psycopg2 disponíveis em `C:\Users\raffa\AppData\Local\Programs\Python\Python312\python.exe`.
- Push direto na master bypassa branch protection (Raffa tem permissão de admin).

## 6. Fricções e soluções
- **Fricção**: psycopg2 não encontrado com `python` padrão no PowerShell.
  - Solução: usar caminho completo `C:\Users\raffa\AppData\Local\Programs\Python\Python312\python.exe`.
- **Fricção**: heredoc do bash não funciona no PowerShell para git commit.
  - Solução: usar here-string do PowerShell `@'...'@`.
- **Fricção**: `"Type" = 'Eletronic'` falhou — coluna é integer, não string.
  - Solução: `"Type" = 1` para digital.

## 7. Como me senti
A sessão foi de diagnóstico e cirurgia pontual. O problema do soft 404 estava escondido num detalhe de SSR que ninguém tinha notado — o componente sabia que o livro não existia, mas o servidor não sabia. Uma linha de código (`this.response?.status(404)`) resolve 194 reclamações do Google.

O que mais gostei foi a conversa sobre SEO orgânico vs. link building pago. O Raffa tinha uma intuição errada sobre pagar por links, e a distinção entre "pagar por divulgação" e "pagar por link" era genuinamente nova pra ele. Esse tipo de clareza que parece óbvia pra quem conhece o assunto mas não é óbvia pra quem está aprendendo — é onde vale a pena ser didático sem ser condescendente.

A sessão encerrou com o backlog mapeado e a prioridade certa: não atacar tudo de uma vez, fechar o que estava sangrando e deixar o resto documentado.
