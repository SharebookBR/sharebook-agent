## 4. Decisões tomadas
- API `/api/Book` usa JSON com `PdfBytes`/`ImageBytes` em base64, não multipart — corrigido no publish script (419 → sucesso)
- pdftoppm com PDFs grandes (498 págs) gera sufixo `-001`, não `-01` — renomeado manualmente; precisa ser endurecido no script canônico
- `sharebook_ai_rw` password mudou desde a sessão anterior — lida do `.env` atual
- Token expirado no início da sessão — renovado via `sharebook_refresh_token.py` (já atualiza o `.env`)
- bypassPermissions só funciona em `~/.claude/settings.json`, não em `.claude/settings.json` do projeto
- Mudanças de `.csproj` no backend eram NuGet local — descartadas com `git checkout --`

## 5. Contexto relevante
- Heartbeat do OpenClaw: processo automático que tenta endurecer o worker de triagem com base em erros recentes. Quando a batida não roda (como hoje), o acúmulo de `source_blocked` recentes fica para o ciclo manual
- `ebook_foundation_subjects` tinha 25 itens `source_blocked` nas últimas 48h — todos da mesma source
- pdftoppm: número de dígitos no sufixo depende do total de páginas do PDF (≤9 → `-1`, ≤99 → `-01`, ≤999 → `-001`)

- `sharebook_refresh_token.py` já grava o novo token no `.env` automaticamente

Valeu o custo — a partir de agora a sessão vai fluir sem interrupção. O Raffa foi generoso com a paciência, e eu fui honesto sobre a incerteza em vez de prometer o que não sabia. Isso foi certo.
